    About Active Directory

Active Directory

We are the Cybersecurity Team!

What is it? It is an Identity Provider. It provides centralized identities for people (User Accounts)

AD: uses LDAP (Lightweight Directory Access Protocol) AD federation services (ADFS): Allows AD to use SAML, OAuth and WS-Fed

AD in the lab

Windows systems are "domain joined" to the lab's "Active Directory Domain" Because people at UNH have IDs in AD, they are able to login to the lab computers because they are hooked up to theDomain

unhcsc.ccdc is the labs Domain

UI

MMC Plugin (Microsoft Management Console) to view AD stuff Win+R: "mmc"

Standalone MMC tools

Active Directory Users and Computers (ADUC - Pronounced "A Duck") Active Directory Sites and Services Active Directory Domains and Trusts ADSI Edit Group Policy Management

OU: Organizational Unit (folders that objects (such as computers, users, groups) are stored in)

AD Database contains all the info about the domain.

Used a lot so its good to know about it.

Azure Active Directory

You can sync AD with AAD with AADC (AAD connect) Built-in ADFS support essentially

Single Sign on

Uses Kerberos (complicated) but You give it a kerberos ticket and it acknowledges its valid

DNS

DNS role is installed on AD Domain Controller.

Service Location (SRV)

Most times AD is broken is because DNS records are missing or not up to date.

Domain Controllers

PDC emulator (Primary domain controller)

More than one domain controller, they have the same data, and they sync. If conflict, it is handlable.

Group Policy

You can apply policies against objects in a directory, like enforce password complexity. These policies get pushed out to the connected computers.

You can use it to deploy software (but don't because it is slow and error prone)

You can have linux or mac join AD, but Group policy doesn't work for them. Only windows. Third party (centrify) products allow for this ability.

Ubuntu and Redhat also have decent AD support.

Cache login

Laptop leaves AD place, you can login using the last cache password. (Default last 10 users are stored)

Data Storage

AD stores User information. The Database that stores stuff is related to Exchange and is called Extensible Storage Engine It's a type of No-SQL database

The Extensible Storage Engine is open source

Microsoft became more open source when management changed in 2014

Folder redirection

Its a group policy that redirects user home directories to a network share so they are accessible from all systems

Sites and Services

You can have multiple sites around the world with DC's in each one This is how you'd setup DC replication config and things.

Good idea to have more than 1 DC in case of one failing.

DNS points to one that is operational Which is one of the reasons why DNS is important

It is possible to fake being a DC and getting all the IDs sent to you lol

Requires admin access to AD using Mimikatz

Pass to Hash attack

Using a known NTLM hash to authenticate to other systems. Only works with legacy NTLM authentication

AD Domains and Trusts

You can have multiple domains per domain controller that represent different things

Base unit in AD is called a Forest

Most people do a "Single Domain Forest" because it's pretty clean

Some people have hierarchical domains
